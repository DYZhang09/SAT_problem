# SAT_problem

## 快速开始
### 运行
直接运行SAT_problem.exe.
### 说明
提供4种模式选择:    
1. cnf解析模式: 输入相应的cnf文件(在 *.\datasets\SAT测试备选算例* 中), 求得满足性等解答写于同名文件中(后缀为 *.res*)
__(求解时请注意内存消耗，内存占用过大时请Ctrl+C退出程序)__
2. 数独求解模式: 输入数独棋盘文件(在 *.\datasets* 中, 文件夹名字对应数独规模), 求得结果打印在屏幕上
3. 数独游玩模式: 随机生成棋盘, 输入难度后显示棋盘, 通过按键进行解答, 可以直接提供答案。
4. debug模式: 输入cnf文件, 打印读取到的公式内容, 求得解答后针对每一个公式进行验算并打印至屏幕
### 数据格式
#### cnf文件
需满足标准的cnf文件格式
#### 数独棋盘文件
有确定值为0或1, 被挖空的格子需为-1, 相邻的数字之间需用空格隔开

## 编译说明
本人不是很懂编译器，自己编译运行该程序可以选择如下两种方式之一：
1. 使用VS2019打开SAT_problem.sln文件，直接编译运行(使用VS其他版本编译可能会报错)
2. 使用DEV-C++ 等等软件直接打开main.cpp编译运行，需要在编译选项中加入指令“-std=c++11 -fpermissive”。编译得到main.exe文件。

_注意_：程序中使用了conio.h中的getch()函数，所以在Linux平台的C编译器下编译可能不成功。

## 文件结构说明
1. binary_puzzle:数独求解与数独游戏模块的代码实现
2. display:主控模块的代码实现
3. naive_dpll, opti_deci_dpll, opti_memo_dpll:cnf求解器实现,分别为原始版本，决策变量优化版本和内存使用优化版本
4. config:配置头文件，各个文件需要的基本库在这里include，同时这里定义数独的阶数和求解器使用的版本。
5. datasets:存放测试样例的文件夹，里面的6,8,10文件夹为对应阶数的数独初始棋盘，剩余的是cnf求解器的样例。
6. puzzleCnfFile:存放求解数独时生成的cnf公式的文件夹。
